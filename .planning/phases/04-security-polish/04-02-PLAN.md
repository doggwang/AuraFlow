<objective>
Make video preview mirroring consistent with placeholder state. When camera is unavailable and placeholder shows, disable the scaleX(-1) transform. When camera is active, keep mirroring for natural self-view.
</objective>

<context>
@D:/Project/vibe coding/AuraFlow/index.html

From index.html lines 125-130 (video element):
```css
#video {
  width: 100%;
  height: 100%;
  object-fit: cover;
  transform: scaleX(-1);
}
```

From index.html lines 132-154 (camera-placeholder):
- Static div, no transforms applied

From index.html lines 1582-1585 (UIController.showCamera):
```javascript
showCamera(active) {
  this.cameraDot.classList.toggle('active', active);
  this.placeholder.style.display = active ? 'none' : 'flex';
}
```

**Issue:** Video is ALWAYS mirrored via CSS (line 129). When placeholder shows, there is no video feed, so the mirror transform serves no purpose and creates inconsistency.

**Solution:** Add a `.mirrored` class to video that applies `transform: scaleX(-1)`, and toggle this class based on camera state. In showCamera(false), remove the mirrored class.
</context>

<task type="auto">
  <name>Task 1: Add mirrored CSS class</name>
  <files>index.html</files>
  <action>
    In the CSS section (around line 125-130), modify the #video rule to remove the hardcoded transform, and instead control mirroring via class:

    Change from:
    ```css
    #video {
      width: 100%;
      height: 100%;
      object-fit: cover;
      transform: scaleX(-1);
    }
    ```

    To:
    ```css
    #video {
      width: 100%;
      height: 100%;
      object-fit: cover;
    }

      transform: scaleX(-1);
    }
    ```

    This separates the transform from the base video style, allowing dynamic control.
  </action>
