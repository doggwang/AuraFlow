From index.html lines 969-992 (ParticleOcean.setupMouseTracking):
```javascript
setupMouseTracking() {
  const onMouseMove = (e) => {
    // Convert screen to world coordinates (approximate)
    const x = (e.clientX / window.innerWidth - 0.5) * 120;
    const y = -(e.clientY / window.innerHeight - 0.5) * 120;
    this.mouseX = x;
    this.mouseY = y;
    this.mouseZ = 0;
  };

  const onTouchMove = (e) => {
    if (e.touches.length > 0) {
      const touch = e.touches[0];
      const x = (touch.clientX / window.innerWidth - 0.5) * 120;
      const y = -(touch.clientY / window.innerHeight - 0.5) * 120;
      this.mouseX = x;
      this.mouseY = y;
      this.mouseZ = 0;
    }
  };

  window.addEventListener('mousemove', onMouseMove);
  window.addEventListener('touchmove', onTouchMove);
}
```

**Analysis:**
- touchmove handler exists and updates mouseX/mouseY
- However, touchmove only fires when finger is ALREADY moving
- Initial touch contact (touchstart) does NOT trigger ripple
- User must drag finger to see effect, but first contact creates no initial ripple

**Fix:** Add a touchstart listener to create an immediate ripple effect when finger first touches the screen. This provides instant feedback on touch.

The vertex shader (line 757-759) uses mouseX, mouseY, mouseZ for ripple:
```glsl
float distToMouse = distance(pos.xy, vec2(uMouseX, uMouseY));
float ripple = sin(distToMouse * 0.5 - uTime * 3.0) * exp(-distToMouse * 0.08) * 3.0;
ripple *= smoothstep(20.0, 30.0, distToMouse);
```

The existing touchmove already sets mouseZ=0, which should work. The issue is touchstart doesn't fire to create initial ripple.

    ```javascript
      if (e.touches.length > 0) {
        const touch = e.touches[0];
        const x = (touch.clientX / window.innerWidth - 0.5) * 120;
        const y = -(touch.clientY / window.innerHeight - 0.5) * 120;
        this.mouseX = x;
        this.mouseY = y;
        this.mouseZ = 1; // Set to 1 to create initial ripple burst
        // Reset to 0 after brief delay for continuous ripple
        setTimeout(() => { this.mouseZ = 0; }, 100);
      }
    };

    window.addEventListener('touchstart', onTouchStart);
    ```

    The mouseZ uniform is used by the shader. Currently it's only set to 0. Setting it to a non-zero value triggers the ripple effect immediately on touch. The timeout resets it to 0 so subsequent touchmove events continue normal behavior.

    Note: The existing touchmove handler (lines 979-988) already sets mouseZ to 0, so the timeout reset is compatible.
